# Solana Playground Feature Inventory

This document lists the implemented features across the full codebase.

## Product Experience Features
- Interactive Solana learning playground for template-based on-chain programs.
- End-to-end learning flow from template discovery to editing, execution, testing, and explanation.
- Developer-focused UX with multiple visual themes and panelized workflows.
- Supports both guided template learning and custom program workflows.

## Frontend Application Features (apps/web)
- Next.js App Router application with route-based pages and layouts.
- Landing pages and marketing flows.
- Login and signup flows.
- Auth callback and auth error handling pages.
- Dashboard for user-oriented navigation and activity.
- Template list and template detail routes.
- Playground index and template-specific playground routes.
- My Code page for user saved code.
- Concepts pages and learning-path-related concept layouts.
- Paths route.
- Sitemap and robots route generation.
- Error boundaries and not-found pages.

## Playground Core Features
- Monaco-based Rust code editor integration.
- Custom editor themes (default, grid, matrix variants).
- Line selection and hover synchronization between panels.
- Breakpoint integration with execution state.
- AI-assisted code completion integration path.
- Auto-save support for custom programs.
- Program session management (template sessions and custom sessions).
- Template opening and switching flows.
- New program creation modal with source selection.
- Template-based new program creation.
- Standard scaffold-based new program creation.
- Practice mode that scaffolds TODO-style learning code.
- Multiple layout modes (code-only, code+map, code+execution).
- Zen mode for focused editing.
- Keyboard shortcuts for navigation and layout switching.
- Command palette (Cmd/Ctrl+K) for actions and template navigation.
- Mobile sidebar open/close flow.

## Sidebar and Layout Features
- Left sidebar with program lists, panel toggles, and template quick-open.
- Desktop-resizable left sidebar with drag handle (VS Code style behavior).
- Sidebar width clamped to min/max bounds for stable UX.
- Sidebar width persistence across reloads via Zustand persistence.
- Default sidebar width increased by ~5% from previous baseline.
- Sidebar-aware loading skeleton width consistency.

## Execution and Simulation Features (UI)
- Execution mode switching between precomputed and live modes.
- Scenario selection and step-based execution triggers.
- Execution logs rendering.
- Before/after account state visualization.
- State diff view and timeline view support.
- Trace rendering hooks.
- Error decoding for execution errors.
- Execution status and badge feedback.
- Transaction builder and account diagram panels for instruction/account context.

## In-Browser Test Experience Features
- Per-program editable test code in Test panel.
- Mocha-style (`describe/it/before/beforeEach/afterEach/after`) browser test runtime with Chai assertions.
- Test run action from panel.
- LeetCode-style pass/fail counters in panel UI.
- Per-test result list with durations and error details.
- Test definition error capture and display.
- Template tests loaded into program sessions for template and template-derived custom programs.
- Default test scaffold auto-seeded for new programs when missing.

## Integrated Terminal Features
- In-app xterm terminal panel docked at bottom of playground.
- Terminal logging for live execution scenarios and custom transactions.
- Terminal logging for test runs.
- Terminal visibility toggled automatically when execution/test events start.
- Buffered terminal output queue in store so logs emitted before terminal mount are flushed on attach.
- Terminal clear action and auto-fit behavior on mount/resize/show.

## Template Test Coverage Features
- Every template has non-generic behavioral test coverage in `packages/solana/templates/*/test.ts`.
- Tests validate instruction presence and method builder availability.
- Tests validate argument names/types and account constraints where applicable.
- Tests validate account schema/state structures where applicable.
- Strong default test scaffold generated by template CLI for new templates.
- Template CLI validation now requires `test.ts` as part of template completeness.
